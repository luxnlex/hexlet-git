Перед тем, как приступить к выполнению практического задания, выполните команду make reset

Если вы сделали что-то не так, и не знаете, как это исправить - вы можете выполнить команду make reset. В результате директория вернется в начальное состояние.

У вас есть проект, который размещается в директории project

В директории выполнены изменения, которые необходимо проиндексировать и создать коммиты

Вам необходимо создать 3 коммита:

Коммит, в котором будут изменения файла .gitignore. Commit message может быть, например, таким: Ignore .sw* files, но вы можте использовать любое другое.
Коммит, в котором будут изменения файла TODO.md. Commit message может быть, например, таким: TODO complete!, но вы можте использовать любое другое.
Коммит, в котором будут изменения всех остальных файлов. Commit message может быть, например, таким: Single quotes replaced by double quotes, но вы можте использовать любое другое.
Как вы будете индексировать изменения и выполнять коммиты - не имеет значения. Цель: создать коммиты с нужным контентом. Предпочтительно, если вы их добавите в порядке 1, 3, 2.

А также:

Попробуйте создать коммит при помощи команд:

git commit
git commit -m "Commit message"
git commit -am "Commit message"
git commit path_to_file
git commit path_to_file -m "Commit message"
Не бойтесь, если после какой-то из этих команд тесты могут не пройти, вы всегда можете выполнить make reset и попробовать снова. Главное, что вы сможете прочувствовать разницу между раными вариантами выполнения команды git commit

После того, как вы выполните 3 требуемых коммита, отправьте решение на проверку.
