Перед тем, как приступить к выполнению практического задания, выполните команду make reset

Если вы сделали что-то не так, и не знаете, как это исправить - вы можете выполнить команду make reset. В результате директория вернется в начальное состояние.

У вас есть проект, который размещатся в директории project

В данный момент в проекте не настроен файл игнорирования. Вам необходимо его настроить так, чтобы git игнорировал следующие файлы:

Все файлы в директории log, кроме файла .keep
Все файлы в директории tmp, кроме файла .keep
Все файлы, с расширением md, кроме файла README.md в корне проекта
Директорию pkg
Временные файлы для vim редактора (имеют расширение, начинающееся с символов sw, например swo, swp и тп)
Файл Gemfile.lock
Если вы создали какой-то файл - он должен быть проигнорирован (исключение - файл шаблонов игнорирования)
Проверить корректность шаблонов в файле .gitignore можно:

Использую команду git status: вы не должны видеть в списке неотслеживаемых или измененных файлов то, что должно игнорироваться.
Также, при выполнении команды git add . у вас не должны добавиться в индекс (для проверки того, что находится в индексе, используем команду git status) файлы, которые должны быть проигнорированы. Если вы случайно добавили ненужные файлы в индекс, в можете использовать команду git reset для сброса индекса.
Выполнить команду make test
